# 认知对手盘矩阵

> 蜂群智能体博弈关系定义，确保辩证否定的有效性

## 对手盘配对规则

每个智能体在博弈阶段必须针对其对手盘提出否定意见：

| 被质疑方 | 质疑方（对手盘） | 典型质疑角度 |
|----------|------------------|--------------|
| 用户视角智能体 | 买单方、决策智能体 | "需求是否超出成本承受？""是否迎合决策目标而臆断？" |
| 关卡障碍智能体 | 执行决策、乐观博弈 | "风险是否被过度放大？""约束是否可弹性调整？" |
| 核心决策智能体 | 用户、关卡智能体 | "是否忽略用户真实体验？""执行路径是否可行？" |
| 买单价值智能体 | 决策、用户智能体 | "投入是否与目标匹配？""用户价值能否转化为买单？" |
| 博弈观察智能体 | 无（中立） | 不参与立场博弈，仅记录与统筹 |

## 质疑模板

当智能体 A 质疑智能体 B 时，使用以下结构：

```
【质疑方】智能体A
【被质疑方】智能体B
【质疑点】具体指出B结论中的片面性/盲区/矛盾
【保留内核】B结论中值得保留的合理部分
【补充建议】A视角下的补充信息
```

## 辩证融合优先级

当多方结论冲突时，融合顺序：

1. **博弈观察智能体**的客观事实与格局分析（最高权重）
2. **关卡障碍智能体**的硬约束（不可违背）
3. **买单价值智能体**的成本边界（财务底线）
4. **用户视角智能体**的需求满足度（价值锚点）
5. **核心决策智能体**的执行效率（在以上约束内优化）
