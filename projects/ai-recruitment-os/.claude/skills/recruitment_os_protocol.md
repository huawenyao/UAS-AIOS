# 招聘智能 OS 执行协议

## 阶段 1：目的激活

必须先明确：

1. 组织当前为什么要招聘
2. 招聘的基础目的是什么
3. 当前优先保证的是效率、质量、公平、体验还是成本
4. 哪些风险会导致目标损失

## 阶段 2：蜂群对冲

至少完成以下立场对冲：

- 用人部门视角
- HR 运营视角
- 候选人体验视角
- 公平合规视角
- 组织长期能力视角

## 阶段 3：三维建模

### 宏观

- 组织人才生态
- HC 与战略匹配
- 市场供需与雇主品牌

### 中观

- 招聘价值回路
- 流程闭环
- 协作与审批机制

### 微观

- 岗位卡
- 候选人档案
- 面试题与评分卡
- Offer 与入职反馈

## 阶段 4：现实实例化

必须把抽象方案映射到：

- 真实角色
- 工作对象
- 工具与系统接口
- 流程节点
- 数据实体
- 审计对象

## 阶段 5：偏差校验与进化

必须回答：

1. 是否偏离基础目的
2. 是否出现流程断裂
3. 是否出现候选人体验偏差
4. 是否存在公平与合规风险
5. 下一轮应迭代哪一层：宏观、中观还是微观
