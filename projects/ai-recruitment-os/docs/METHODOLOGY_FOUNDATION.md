# 招聘智能 OS 方法论基础

## 方法论来源

本项目同时继承两套方法：

1. **蜂群智能体方法**
   - 用于解决招聘中的多立场博弈
   - 对冲用人部门、HR、候选人、公平合规、组织长期能力等冲突

2. **三维理念现实智能体群方法**
   - 宏观：组织人才生态、市场供需、雇主品牌
   - 中观：招聘价值回路、评估闭环、协作审批流程
   - 微观：岗位、候选人、题目、评分卡、Offer、入职反馈等真实对象

## 基础目的

招聘智能 OS 的基础目的不是“单纯提高效率”，而是：

**让组织在可控成本下，以更高确定性、更低偏差、更好体验，持续获得与组织阶段真正匹配的人才。**

## 为什么采用演化开发

招聘系统容易出现：

- 目标漂移
- 候选人体验偏差
- 面试官评分口径失衡
- 组织流程断裂
- 合规与公平风险

因此项目要求：

1. 先目的激活
2. 再蜂群对冲
3. 再三维建模
4. 再现实实例化
5. 再偏差校验
6. 再进化规划

## 项目内对应关系

| 方法论要求 | 项目实现 |
