## 一、产品愿景与战略定位

### 1\.1 一句话定义

**AIOS = 企业智能经营操作系统，让「数据分析 → 策略决策 → 行动执行 → 结果归因 → 策略迭代」的智慧闭环从人脑隐含变为系统显式。**

### 1\.2 核心命题

企业最宝贵的资产不是数据，而是**决策智慧** — 即"为什么做这个决策、这个决策带来了什么结果、下次应该怎么调整"的完整认知链。

但今天这个认知链是**断裂的**：

```Plain Text
当前断裂的认知链：
                                                    
  经营分析会        定 Todo          执行           结果
  "Q2 Margin       "各BD本周        做了...        Margin
   差了3%"   →     拜访5个客户" →    但不知道     → 涨了还是跌了
                                   哪些有效        不知道归因
      ↓               ↓               ↓              ↓
   有数据          有动作           无追踪          无归因
   有分析          有指派           无反馈          无迭代
                                                    
   ── 人脑隐含 ──  ── 飞书任务 ──  ── 黑箱 ──  ── 断裂 ──

```

**AIOS 要修复这条链**：

```Plain Text
AIOS 修复后的认知链：

  数据感知     智能策略     精准行动     结果归因     策略进化
  ──────→    ──────→    ──────→    ──────→    ──────→
  Margin      根因是       BD-A 执行    BD-A 的     该策略在
  跌了3%      TR下降       策略X        动作带来     此赛道有效
              + CFC客户    策略Y        +2% Margin   沉淀为
              流失         策略Z        BD-B 的      SOP
                                     策略Y无效
      
  ── AI检测 ── ── AI归因 ── ── AI分派 ── ── AI归因 ── ── AI学习 ──

```

## 二、问题空间：为什么现在需要一个 AIOS

### 2\.1 企业经营的核心痛点

|\#|痛点|现状|根因|
|---|---|---|---|
|1|**决策智慧隐含**|经营分析会的讨论、归因逻辑、策略选择理由全靠会议纪要和个人记忆|没有系统记录"为什么这样做"|
|2|**行动归因断裂**|定了 Todo 但不知道哪个 Todo 带来了结果变化|行动与指标没有关联追踪|
|3|**策略无法复用**|上个季度验证有效的策略，下个季度换了人就不知道了|没有策略库 \+ 效果评估|
|4|**PDCA 执行衰减**|Plan 定得好，Do 也做了，但 Check 和 Act 流于形式|缺乏自动化 Check \+ 结构化 Act|
|5|**数据到决策的链路不透明**|分析师出的报告→管理层拍脑袋→执行层不理解|缺少从数据洞察到策略建议的推理链|

### 2\.2 市场机会

**AI 时代的企业经营不是缺数据，是缺"数据到决策到归因"的闭环系统。**

```Plain Text
当前市场格局：

  BI 工具（Quick BI / Superset）     → 解决"看数据"
  OKR 工具（飞书 OKR / Lattice）     → 解决"定目标"
  项目管理（飞书项目 / Jira）         → 解决"管任务"
  CRM（Salesforce / HubSpot）        → 解决"管客户"
  
  ❌ 没有人解决"数据 → 策略 → 行动 → 归因 → 迭代"的闭环
  ❌ 没有人把"经营分析会的智慧"变成可复用的系统资产
  ❌ 没有人让"为什么这样做有效"变成机器可学习的知识

```

## 三、产品定义：AIOS 是什么

### 3\.1 产品本质

**AIOS 不是又一个 BI 工具，也不是又一个项目管理工具。**

**AIOS 是企业第一个"经营智慧操作系统"** — 它让每一次经营决策的完整逻辑链（感知→归因→策略→行动→归因→进化）变成可追踪、可学习、可复用的数字资产。

### 3\.3 五大核心实体

|实体|定义|关键属性|
|---|---|---|
|**Objective 目标**|经营目标的结构化表达|KPI指标、目标值、时间窗口、组织归属、分解关系|
|**Strategy 策略**|达成目标的方法论假说|归因假设、策略方案、预期效果、置信度、来源\(人/AI\)|
|**Action 行动**|策略的具体执行|任务描述、负责人、SLA、状态机、关联策略|
|**Result 结果**|行动的产出度量|实际值、达成率、偏差量、归因标记|
|**Attribution 归因**|连接行动与结果的因果推断|贡献度、因果方向、置信区间、归因方法|

**核心创新**：Strategy 和 Attribution 是 AIOS 独创的实体 — 它们是"经营智慧"的结构化载体。

### 4\.2 六大 AI 引擎详述

#### 引擎 1：感知引擎 \(Sense Engine\)

**职责**：自动发现经营数据和组织运行中的信号。

|能力|输入|输出|技术|
|---|---|---|---|
|异常检测|四大金刚时序数据|异常事件（指标偏移/趋势反转）|Z\-score \+ 同比偏移 \+ LLM辅助|
|趋势预测|历史数据 \+ 季节因子|QTD预测 \+ 置信区间|线性外推 \+ Prophet|
|机会发现|赛道/客户/产品矩阵|增长机会（高增速低渗透）|SOC/SOM扫描 \+ 交叉销售分析|
|政策感知|媒体政策变更信息|政策窗口机会/风险|NLP提取 \+ 规则匹配|

#### 引擎 2：归因引擎 \(Attribution Engine\)

**职责**：回答"为什么"和"谁的贡献"。

|能力|输入|输出|技术|
|---|---|---|---|
|维度下钻归因|指标变化量|各维度贡献度排序|自动GROUP BY \+ 贡献度计算|
|行动归因|行动记录 \+ 指标变化|行动→结果的因果贡献|反事实推断（A/B对比）|
|策略归因|策略执行记录 \+ 结果数据|策略有效性评估|统计检验 \+ 置信度|
|多因素归因|多变量同时变化|各因素贡献拆分|Shapley值 \+ 结构归因|

**核心创新**：行动归因 — 这是市面上没有产品做到的。将"BD做了什么"与"指标发生了什么变化"做因果关联。

#### 引擎 3：策略引擎 \(Strategy Engine\)

**职责**：基于归因结果，生成和推荐策略方案。

|能力|输入|输出|技术|
|---|---|---|---|
|策略推荐|归因结果 \+ 策略库|Top\-N策略方案（含预期效果）|相似度匹配 \+ 历史效果加权|
|方案生成|目标 \+ 约束 \+ 资源|行动计划（Who/What/When）|LLM生成 \+ 模板约束|
|效果预估|策略方案 \+ 历史数据|预期指标变化量|回归模型 \+ 类比推断|
|策略评估|策略执行结果|策略评分（有效/无效/待验证）|假设检验 \+ 贝叶斯更新|

#### 引擎 4：编排引擎 \(Orchestration Engine\)

**职责**：把策略变成可执行的行动，协调人和 Agent。

|能力|输入|输出|技术|
|---|---|---|---|
|任务分派|策略方案 \+ 组织架构|工单（负责人/SLA/验收标准）|OMTU匹配 \+ 能力匹配|
|Agent协同|复杂任务分解|Agent子任务编排|DAG调度 \+ 5Agent协同|
|进度跟踪|工单状态变更|进度看板 \+ 超时预警|状态机 \+ SLA计时|
|闭环确认|行动完成信号|触发归因验证|自动触发归因引擎|

#### 引擎 5：预警引擎 \(Alert Engine\)

**职责**：全链路监控和升级处理。

|能力|输入|输出|技术|
|---|---|---|---|
|指标预警|KPI偏差超过阈值|预警通知 \+ 建议动作|动态阈值 \+ 分级|
|SLA预警|任务超时/即将超时|升级通知|计时器 \+ 升级规则|
|策略预警|策略执行效果偏离预期|策略调整建议|效果跟踪 \+ 止损线|
|闭环预警|PDCA链路断裂|补链通知|状态完整性检查|

#### 引擎 6：进化引擎 \(Evolution Engine\)

**职责**：让系统越用越聪明。

|能力|输入|输出|技术|
|---|---|---|---|
|SOP沉淀|验证有效的策略\+行动|标准操作流程（SOP）|模式提取 \+ LLM总结|
|策略淘汰|多次验证无效的策略|标记为不推荐|贝叶斯置信衰减|
|知识图谱|策略\-行动\-结果的关联|经营知识图谱|图构建 \+ 关联挖掘|
|场景学习|不同场景的效果差异|场景适配规则|决策树 \+ 条件推理|

## 五、核心场景：AIOS 怎么用

### 5\.1 场景一：经营分析会的 AI 增强

**传统模式**：

```Plain Text
人工准备数据 → 会上讨论归因 → 口头定策略 → PMO记Todo → 执行黑箱 → 下次会再看数

```

**AIOS 模式**：

```Plain Text
AI自动检测异常 → AI预归因（贡献度排序）→ AI推荐策略方案 → 系统生成Action
→ 自动跟踪执行 → 7天后自动归因验证 → 有效的沉淀为SOP

```

**具体流程**：

|步骤|传统（人驱动）|AIOS（AI驱动）|效率提升|
|---|---|---|---|
|会前准备|分析师3天出报告|感知引擎自动推送异常\+预归因|3天→10分钟|
|归因分析|会上拍脑袋讨论|归因引擎给出维度贡献度排序|主观→客观|
|策略制定|经验驱动|策略引擎从策略库推荐Top方案|经验→数据|
|任务分派|PMO手动创建任务|编排引擎自动生成Action\+SLA|2小时→自动|
|执行跟踪|PMO催、群里问|预警引擎自动监控\+升级|被动→主动|
|效果验证|下次会再看|归因引擎7天后自动验证|断裂→闭环|
|经验沉淀|依赖个人记忆|进化引擎自动沉淀SOP|隐含→显式|

### 5\.2 场景二：日常经营巡检

```Plain Text
每日 09:00，感知引擎自动执行：
  │
  ├── 检查四大金刚：Spend/Revenue/Margin/TR 是否有异常偏移
  │
  ├── 发现异常 → 自动归因（哪个BU/客户/赛道贡献了变化）
  │
  ├── 归因完成 → 策略引擎推荐应对方案
  │
  ├── 方案确认 → 编排引擎生成Action派发给负责人
  │
  ├── 负责人执行 → 系统跟踪 → 7天后自动归因验证
  │
  └── 验证结果 → 有效→SOP沉淀 / 无效→策略淘汰

```

### 5\.3 场景三：季度策略迭代

```Plain Text
每季度，进化引擎自动生成「策略进化报告」：
  │
  ├── 本季度执行了哪些策略？各策略的验证结果？
  │
  ├── 哪些策略被验证有效？→ 升级为SOP
  │
  ├── 哪些策略被验证无效？→ 标记淘汰 + 原因分析
  │
  ├── 哪些场景还没有覆盖？→ 下季度策略空白点
  │
  └── 经营知识图谱更新 → 策略-行动-结果的因果关系更清晰
```

## 六、产品路线图

### 6\.1 MVP 定义（M1\-M2 · 8周）

**MVP 目标**：验证"数据→策略→行动→归因"闭环在一个 BU 跑通。

|模块|MVP 范围|不做|
|---|---|---|
|感知引擎|四大金刚异常检测 \+ 飞书推送|趋势预测、机会发现|
|归因引擎|维度下钻归因（自动GROUP BY）|行动归因、多因素归因|
|策略引擎|人工定义策略 \+ 结构化记录|AI推荐、效果预估|
|编排引擎|飞书任务自动创建 \+ SLA|Agent协同、复杂编排|
|预警引擎|指标阈值预警 \+ SLA超时|策略预警、闭环预警|
|进化引擎|策略效果手动标记|自动SOP沉淀、知识图谱|

**MVP 验证指标**：

|指标|目标值|度量方式|
|---|---|---|
|异常检测准确率|\> 80%|推送异常中真异常占比|
|归因分析覆盖率|\> 70%|异常事件中自动归因成功占比|
|Action闭环率|\> 60%|创建的Action中有结果验证的占比|
|经营分析会准备时间|降低 80%|从3天→\<半天|

### 6\.2 Phase 2：智能化（M3\-M6）

|能力|升级内容|
|---|---|
|感知引擎|\+ 趋势预测 \+ 机会发现（百路摸排）|
|归因引擎|\+ 行动归因（连接Action与Result） \+ 多因素归因|
|策略引擎|\+ AI策略推荐 \+ 效果预估|

|进化引擎|\+ 自动SOP沉淀 \+ 策略淘汰机制|

### 6\.3 Phase 3：自治化（M7\-M18）

|能力|目标|
|---|---|
|全链路自治|感知→归因→策略→行动→验证→进化 全自动化|
|经营知识图谱|策略\-行动\-结果的因果关系网络|
|跨BU学习|A\-BU验证有效的策略自动推荐到B\-BU|
|行业模型|出海行业的经营决策模型微调|

## 九、关键成功因素

|\#|因素|为什么重要|
|---|---|---|
|1|**管理层真用**|AIOS 的价值在于闭环，管理层不用就没有策略输入|
|2|**归因引擎做准**|归因是 AIOS 的核心差异点，做不准就沦为又一个 BI|
|3|**策略库做厚**|策略库是 AIOS 的护城河，空库 = 无推荐能力|
|4|**数据底座做稳**|没有统一口径，一切度量都是空中楼阁|
|5|**从一个BU打透**|不要一上来全公司铺开，先在 PAC 跑通再复制|

## 十、风险与应对

|风险|概率|影响|应对|
|---|---|---|---|
|归因准确率不足|高|高|先做维度下钻归因（成熟技术），行动归因作为Phase 2|

|数据质量拖后腿|高|中|Phase 1 聚焦已有高质量数据（宽表\+GMC报表）|
|策略库冷启动|高|中|从历史经营分析会的决策记录中提取种子策略|
|变成又一个看板|中|高|严守闭环设计：每个Action必须关联Strategy，每个Result必须关联Attribution|

## 十一、核心理念总结

```Plain Text
AIOS 不是一个工具，是一种经营哲学：

  "企业最宝贵的资产不是数据，是决策智慧。"
  
  "决策智慧 = 为什么做 + 做了什么 + 效果如何 + 下次怎么改"
  
  "AIOS 让这条链从人脑隐含 → 系统显式 → 机器可学习 → 组织可进化"

```

```Plain Text
AIOS 的信仰：

  数据驱动 ≠ 智能经营
  数据驱动只是起点
  智能经营 = 数据感知 + 智能归因 + 策略推荐 + 精准行动 + 结果归因 + 策略进化
  
  这个闭环每转一圈，企业就聪明一分。
  这就是 AIOS 的终极价值：让企业越经营越聪明。

```
