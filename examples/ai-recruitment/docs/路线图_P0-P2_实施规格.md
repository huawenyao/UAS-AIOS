# AI 招聘系统 · P0–P2 实施规格

> 承接「内外部洞察」结论，将最大优化价值拆解为可验收交付物。  
> 约束：不修改 `acceptance/`；新增能力以 `scripts/`、`docs/`、`run_batch.py` 为主。

---

## 一、北极星与成功标准

| 指标 | 定义 | P0 目标 | P2 目标 |
|------|------|---------|---------|
| TTF 贡献 | 初筛→短名单可决策时间 | 批次 < 10 分钟（100 份以内） | 与 ATS 事件对齐可度量 |
| 面试官有效小时/录用 | — | 不考核（缺录用回流） | Learn 阶段接入 |
| 短名单精度@k | Top-k 进入终面比例 | 人工抽检记录 | 自动对比 evaluation |
| 可解释决策率 | 带 evidence 的 recommend 占比 | ≥ 95% | 100% + 公平审计包 |
| 候选人体验 | 反馈 SLA | HR 可复制拒信话术 | 候选人门户/邮件 |

## 二、P0（已启动落地）

### 2.1 价值摘要契约

- **模块**: `scripts/value_summary.py`
- **API**:
  - `build_value_summary(results, ...) -> dict`
  - `format_value_summary_console(summary) -> str`
  - `render_value_banner_html(summary) -> str`
- **输出字段**: `headline`, `valid_count`, `decisions`, `interview_ready_count`, `explainability_rate`, `estimated_saved_minutes`

### 2.2 一键入口

- **入口**: `python run_batch.py [--resume-dir DIR] [--feedback]`
- **链路**: `workflow_execution` → `value_summary.json` → `closed_loop_last.json` → 打开 `recommendation_list.html`
- **角色**: HR 无需 Cursor/多命令记忆

### 2.3 推荐名单首屏

- **文件**: `scripts/generate_recommendation_list.py`
- **要求**: 页面顶部展示 `value-banner`（一句话 + 四指标）

### 2.4 验收（不扩 acceptance）

- 手工：`cd examples/ai-recruitment && python run_batch.py --resume-dir test_fixtures`
- 检查：`reports/value_summary_*.json`、`reports/recommendation_list.html` 首屏文案
- 回归：`python -m pytest acceptance/ -v`

## 三、P1（下一迭代）

### 3.1 候选人反馈（已有脚本，需接入主路径）

| 项 | 说明 |
|----|------|
| 模块 | `scripts/candidate_feedback.py` |
| 触发 | `run_batch.py --feedback` 或 `run_interactive` 菜单项 |
| 产出 | `reports/feedback/rejection_feedback_*.json` |
| 合规 | 话术脱敏、HR 人工审核后发送；文档见 `docs/候选人反馈使用指引.md`（待建） |

### 3.2 面试闭环可见性

| 项 | 说明 |
|----|------|
| 实体 | Task `ai_interview` → Event `ai_interview_completed` |
| 运行时 | `scripts/entity_runtime.py` |
| UI | `generate_status_panel.py` 标出「已通知 / 已评价」 |
| 缺口 | `/evaluate` 与 LLM 面试题生成 → P2 |

### 3.3 角色视图（轻量）

| 角色 | 首屏关注点 |
|------|------------|
| HR | 批次摘要、推荐/待定人数、耗时 |
| 招聘经理 | 决策分布、强推名单 |
| 面试官 | 单人报告「风险 + 证据 + 建议问题」（`report_renderer` 扩展一节） |

## 四、P2（战略扩展）

### 4.1 AI 结构化初面

- 异步问答（文本先行）→ 可选 RTC（参考 AI-HR / HireVue）
- 与 `Task.type=ai_interview` 对齐

### 4.2 ATS / 渠道集成

- Webhook：Moka / Greenhouse 只读候选人 + 回写 `decision` + `scores`
- 国内：Boss 直聘 API 或 RPA 插件（ADR：能力先于 Agent，见 `harness/knowledge/constraints/adr-edh-002`）

### 4.3 Learn 闭环

```text
Evaluation.hire_outcome (hired / rejected_after_interview / left_probation)
    → 更新 evaluation_criteria 权重建议
    → 人工批准 → configs 版本化
```

### 4.4 多 Agent 网格（UAS-AIOS）

| Agent | 能力服务 ID（建议） |
|-------|---------------------|
| Profiler | `cap.recruitment.jd_parse` |
| Screener | `cap.recruitment.resume_score` |
| Interviewer | `cap.recruitment.ai_interview` |
| Communicator | `cap.recruitment.candidate_notify` |
| Orchestrator | `cap.recruitment.closed_loop` |

编排由 `recruitment_closed_loop` 状态机 + 人在环门（外联/Offer）实现。

---

## 五、与竞品差异化的「不做清单」

- 不做完整 ATS 替代（与 Moka/Greenhouse 共生）
- 不做无证据黑箱排序
- 不做全自动 Offer / 淘汰邮件（仅生成话术，人工发送）
- 不优先视频情绪分析（合规与 ROI 低）
