# 候选人反馈使用指引（P1）

## 用途

为 **未通过初筛** 的候选人生成脱敏、可解释的反馈话术，供 HR **人工审核后** 复制到拒信或招聘系统中。不自动发送邮件。

## 生成方式

```bash
# 需先跑批次产生 database/candidates.json
python run_batch.py --resume-dir ./test_fixtures --feedback

# 或单独运行
python scripts/candidate_feedback.py --batch
```

产出路径：`reports/feedback/rejection_feedback_YYYYMMDD_HHMMSS.json`

## 合规要点

1. **人工审核**：AI 话术仅供参考，发送前 HR 必须核对事实与语气。  
2. **脱敏**：不包含其他候选人信息或内部评分规则细节。  
3. **公平说明**：可附带「评估维度公开、结果可审计」类说明（见 `docs/产品价值感设计.md` §4.3）。  
4. **欧盟/高风险 AI**：若面向 EU 候选人，需对照当地就业 AI 透明度要求单独法务确认。

## 内容来源

- `risk_flags` → 中文风险标签  
- `evidence` → 缺口描述（无 risk 时回退）  
- `scores.total_score` → 可选展示综合分（对外可改为区间表述）

## 与价值闭环

候选人侧体验指标（`candidate_experience_score`）在录用结果回流（P2 Learn）前，以「是否有可发送话术」作为过程指标。
