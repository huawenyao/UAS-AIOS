# selfpaw 原生认知智能体蜂群（验证项目）

## 系统概述

本项目验证如何用 selfpaw 原生蜂群替代单一主观决策：用户、关卡、决策、买单、观察五个智能体分别构成独立认知单元，经由“第一次否定 → 第二次否定 → 辩证融合”产出最终方案。

## 核心命令

- `/dialectic [议题]` - 发起完整蜂群分析
- `/challenge [议题ID]` - 对当前结论执行第二次否定
- `/retrospective [议题ID]` - 基于结果复盘蜂群判断

## 工作流

1. **议题归一化** - 明确目标、边界、资源和成功标准
2. **第一次否定** - 五个智能体并行输出独立报告
3. **第二次否定** - 对手盘交叉质询，暴露盲区与冲突
4. **辩证融合** - 观察智能体统筹共识与冲突，生成最终方案
5. **结构化落盘** - 报告脚本写入 `database/decisions/` 与 `reports/`

## 知识层

| 文件 | 用途 |
|------|------|
| `configs/workflow_config.json` | 蜂群工作流定义 |
| `configs/swarm_agents.json` | 智能体角色、使命、对手盘定义 |
| `.claude/skills/swarm_protocol.md` | 三阶段执行协议 |
| `.claude/skills/decision_output_contract.md` | 最终输出契约 |

## 验证标准

- [x] 多主体：五大维度均有独立立场和交付物
- [x] 对手盘：存在明确质询关系与冲突暴露机制
- [x] 升维锁定：最终结论以蜂群融合结果为准
- [x] 可审计：决策 JSON 和 Markdown 报告可追溯
