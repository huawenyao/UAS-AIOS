## 1. 一句话定位（按层级）

| 层级 | 一句话 |
|------|--------|
| 决策层 | 可追责的数字岗位，换系统不用推倒 Agent |
| 信息化 | Agent 集成的企业总线：N Agent → 1 套 cs → M 系统 |
| 业务负责人 | 带 KPI 的数字岗位，能办的事在清单里 |
| 一线员工 | 懂流程的同事：代填代查，搞不定自动升级 |
| 合规 | 所有写库走 cs，全链 audit，红线 deterministic |

---

## 2. 能力评审会议程（2 小时）

| 时间 | 议题 | 产出 |
|------|------|------|
| 0:00 | 闭环场景确认（如留资→报价） | 场景说明书 1 页 |
| 0:20 | 列出 8–12 个业务动词 | 能力卡片 draft |
| 0:50 | scope / 红线 / deterministic 标注 | 评审签字 |
| 1:10 | 连接器与 SoR 确认 | backend 分配 |
| 1:30 | 岗位白名单与 KPI | ΠPaw 编制表 |
| 1:50 | R0 里程碑与验收 FR | 项目计划 |

---

## 3. R0 试点包（可直接立项）

- **场景**：B2B 留资 → 资格 → 报价草案 → 审批  
- **FR 包**：见 EPS-02 §7  
- **技术**：EPS-03 R0 任务 T0-1～T0-6  
- **周期验收**：10 条真实线索端到端 + 10 条 audit 可回放  

---

## 4. 反对意见速查

| 反对 | 回应 |
|------|------|
| 和 Copilot 重复 | Copilot 无岗位白名单与 cs 审计 |
| 又一套中台 | cs 是动词层，不存全量业务数据 |
| 实施太重 | R0 仅 8 个能力 + 1 个连接器 |
| 员工不用 | 只开放 3 个 cs，用「少点几次鼠标」衡量 |
