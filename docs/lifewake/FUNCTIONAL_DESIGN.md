# 生命回响 · 功能设计（MVP）

> 聚焦两条核心体验，确保最小可行性与情感冲击；扩展功能只预留接口。

---

## 1. 目标用户与启动策略

| 群体 | 价值主张 | v0.1 标杆场景 |
|------|----------|---------------|
| 情侣（含异地） | 思念可听、可感、可纪念 | 双人心跳共鸣交响曲 |
| 创作者 | 被理解的灵感礼物 | 惊喜盲盒歌曲/画作/灵感任务 |

启动策略：邀请制内测 → 情感敏感用户 → 每个功能上线前通过「情感冲击测试」。

---

## 2. 核心功能一：「惊喜盲盒」—— 潜意识的回音壁

### 2.1 能力目标

AI 在用户授权范围内学习低敏感「隐性信号」，在无预期但合适的时机推送量身定制惊喜，并附上灵感来源解析。

### 2.2 惊喜类型（v0.1 mock）

| 类型 | 输入信号 | 输出 |
|------|----------|------|
| `song` | 哼唱旋律片段 + 偏好歌手风格 | mock 音频资产 + 歌词摘要 |
| `artwork` | 近期照片风格标签 + 潜意识关键词 | mock 图像资产 + 风格说明 |
| `inspiration_task` | 地理位置粗粒度 + 心情 | 可执行线下微任务 |

### 2.3 用户流程

1. 用户明确同意低敏感数据范围（可随时撤回）。
2. AI 后台静默分析，不主动打扰。
3. 在时机窗口（清晨 / 通勤 / 无聊信号）推送盲盒。
4. 展示 `inspiration_trace`：「你上周反复听爵士，所以这首融合了你的哼唱」。
5. 用户可收藏、分享（需二次确认）、反馈 wow / meh。

### 2.4 成功标准

- 首次体验 `wow_score >= 0.7`
- 每条惊喜可追溯到 ≥1 条用户特有信号
- 无同意或撤回后不得推送

## 3. 核心功能二：「心跳音乐」—— 生理数据转化为情感共鸣

### 3.1 能力目标

将实时（或会话内）心跳节奏转化为动态音乐；支持单人「生物韵律」与双人「共鸣交响曲」。

### 3.2 模式

| 模式 | 输入 | 输出 |
|------|------|------|
| `solo` | 单人 PulseStream + 风格偏好 | 生物韵律音乐 + 波形可视化 |
| `duet` | 双方 PulseStream + 双方同意 + Bond | 共鸣交响曲 + 同步可视化 + 纪念音频引用 |

### 3.3 用户流程

1. 连接并授权心跳监测设备（mock device link）。
2. 选择单人或邀请伴侣加入。
3. 设置音乐风格（classical / electronic / nature）。
4. 实时（会话级 mock）生成音乐；可调心跳与背景音比例。
5. 结束会话可保存为纪念物；双人共享需双方同意。

### 3.4 成功标准

- 单人模式可完成一次会话并产出可回放资产引用
- 双人模式必须双方 consent + Bond Guardian 双向需求通过
- 任一方撤回共享 → 共享链接失效，本地纪念物仍属各自主权

## 4. 扩展功能接口（不实现执行路径）

| 功能 | 预留能力 ID | 说明 |
|------|-------------|------|
| 记忆时光机 | `lw.memory.weave` | 指定日期照片/文字/位置 → 互动叙事 + 配音 |
| 关系纽带 | `lw.bond.visualize` | 互动频率、话题云、情感升温任务 |
| 数字孪生对话 | `lw.twin.draft` | 语言风格分身代回；必须用户审核后发送 |

扩展能力在 `system_registry` 中标记 `status: reserved`，调用返回 `FEATURE_RESERVED`。

## 5. Agent 编织（功能视角）

| Agent | 职责 | 主要交付 |
|-------|------|----------|
| Signal Weaver | 从授权信号中提取独特模式 | SignalBundle |
| Surprise Alchemist | 将信号炼成惊喜作品 | SurpriseArtifact |
| Pulse Composer | 心跳 → 音乐映射与混音 | PulseComposition |
| Bond Guardian | 双向情感需求与共创归属 | BondCheckResult |
| Privacy Steward | 同意、撤回、最小化、用途限制 | ConsentDecision |
| Ritual Host | 包装为自我确认仪式 | RitualView |
| Evolution Listener | 收集 wow/meh → ChangeSet | ChangeSetDraft |

## 6. 体验原则（功能层检查清单）

- [ ] 是否让用户感到「被理解」而非「被分析」？
- [ ] 是否附带灵感来源解析？
- [ ] 是否尊重慢灵感（非刷屏推送）？
- [ ] 关系功能是否双向？
- [ ] 是否可通过情感冲击测试？
