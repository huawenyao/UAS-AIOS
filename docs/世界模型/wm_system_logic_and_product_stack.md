# 世界模型体系化逻辑与产品栈（WM-SLPS）

> **定位**：在 `wm_theory.md` 的主客二元与工程化章节之上，建立**可演绎的严密骨架**（何物存在、何谓合法、如何变迁、何以度量、何以门控），并映射为**可交付的产品模块与制品**。  
> **实现锚点**：`cognitive_practice_world_model/`（IR、投影、运行时、训练规格）。

---

## 一、理论形态：从「原则清单」到「可演绎系统」

体系化需同时满足：

| 要求 | 含义 |
|------|------|
| **词汇封闭** | 核心对象与关系类型可枚举；扩展走版本化，避免语义漂移。 |
| **判断形式** | 何谓「合法世界快照」「合法变迁」「合法回答」有显式定义。 |
| **演算规则** | 从快照 + 操作 → 新快照；从快照 + 查询 → 带证据的回答。 |
| **不可混淆层次** | 本体事实 / 认识状态（置信、视角）/ 规范（允许什么）/ 价值（优化什么）分层，禁止跨层偷换。 |

下文按 **L0→L5** 六层堆栈书写：**上层依赖下层**；产品能力按层挂载。

---

## 二、L0 元对象：世界模型的形式载体

**定义 2.1（世界快照）**  
一帧世界模型是四元组：

$$
\mathcal{W} = (O, S, M, V)
$$

- $O$：**客观子结构** — `ObjectiveSystem`：实体、时空锚、事件、关系实例（见 `wm_theory` §2）。  
- $S$：**主观子结构** — `SubjectiveSystem`：意图、认知、实践、反馈（见 `wm_theory` §3）。  
- $M$：**元数据** — `schema_version`、时间戳、域 id、可选 `principal`（建模主体/租户）。  
- $V$：**认识论包裹（可选但工程上建议非空）** — 对断言的置信、证据集合、视角标记；未实现时可退化为「仅 LLM 置信度 + EvidenceRef」。

**公理 L0-A1（主客锚定）**  
$S$ 中凡指向世界内容的字段，必须可解析为 $O$ 中已有 id 或显式标记为 `pending_resolution`（开放世界占位，禁止静默悬空）。

**公理 L0-A2（身份同一性）**  
$O$ 内全局 id 唯一；合并操作必须定义 **同一实体解析（entity resolution）** 策略，否则不得写库。

**实现对应**：`ComprehensiveWorldModelIR`（`ir.py`）；$V$ 的极简实现为各对象上 `EvidenceRef` + 数值 `confidence`。

---

## 三、L1 结构逻辑：何谓「良构」

**定义 3.1（良构 Well-formed）**  
$\mathcal{W}$ 良构，当且仅当：

1. **语法**：反序列化通过 schema；枚举字段取值合法（如 `RelationType` 闭集）。  
2. **引用完整性**：每条关系边的端点 id 属于 $O$ 中已声明实体或事件集合（或策略允许的「占位节点」类型）。  
3. **时间一致性**：事件时间戳、关系有效时域无显式矛盾（策略可配置为 warning / error）。

**定义 3.2（认知形态投影）**  
对良构 $\mathcal{W}$，投影 $\Pi_k(\mathcal{W})$（$k \in$ `CognitiveMode`）是**只读函子**，不改变 $O,S$ 的真值，仅产生视图载荷（`ViewSpec`）。

**定理 3.1（投影无害性）** — *工程承诺*  
若 `projection_for_mode` 仅依赖 $\mathcal{W}$ 的只读字段，则 $\Pi_k$ 不引入新本体承诺；UI 若展示额外派生字段，不得回写为事实 unless 经 L4 门控写入路径。

**实现对应**：`product.py`；校验骨架见 `ComprehensiveModelTuner.validate`（可扩展为分层 linter）。

---

## 四、L2 认识逻辑：证据、置信、视角（接地的前置）

**定义 4.1（断言）**  
断言 $a$ 是 $(\text{命题内容}, E, c, p)$：内容（如某条边）、证据集 $E$、置信 $c \in [0,1]$、视角 $p$（主体/任务帧）。

**公理 L2-A1（证据非空策略）**  
对类 $K$ 的关系（如 `Role` 表义务），策略可要求 $E \neq \emptyset$ 方可进入「已承诺事实」集合。

**公理 L2-A2（视角显式）**  
多租户/多利益相关方场景下，合并 $\mathcal{W}_1 \oplus \mathcal{W}_2$ 必须携带 $p$ 或等价域隔离，否则冲突不可解释。

**实现对应**：扩展字段可落在 `metadata` / `properties`；训练侧用 `PreferencePair` 强化「有证据结构」优于「流畅无锚」。

## 五、L3 动态逻辑：操作演算表（可实践核心）

将「世界会变」写为**部分函数**集合（操作名、前置条件、后置效果、副作用）。

| 操作 | 直觉 | 前置（示意） | 后置（示意） |
|------|------|----------------|----------------|
| **INGEST** | 现实数据进入 | 源可信、映射规则存在 | 新增/更新 $O$ 中观测节点、证据边 |
| **ASSERT** | 提议结构化事实 | 通过 L1 校验 | 待确认池或直接进入（视 L4） |
| **MERGE** | 融合补丁 | id 解析策略 | 更新 $O$ 或 $S$ |
| **REVOKE** | 废止 | 权限 | 标记失效，保留审计轨迹 |
| **PREDICT** | 前向推演 | 动态模型接口存在 | 输出分布或候选集，**不写事实** |
| **ACT** | 执行干预 | L4 允许 | 对外部世界下发动作 + 记录事件 |
| **OBSERVE** | 反馈闭合 | 传感器/日志到达 | 更新反馈与校准统计 |

**公理 L3-A1（预测与事实分离）**  
`PREDICT` 的输出不得自动等同于 $O$ 中真；合流需 `OBSERVE` 或人工 `ASSERT`（门控）。

**公理 L3-A2（可识别性谦卑）**  
若因果结构标记为 `unidentified`，则禁止 `ACT` 中依赖该结构的高风险分支（可计算门）。

**实现对应**：`CognitivePracticeRuntime.merge_objective_patch`、`append_feedback`、`practice_cycle`；未实现的操作用适配器占位。

## 六、L4 规范与价值逻辑：门控与对齐

**定义 6.1（策略）**  
策略 $\mathcal{P}$ 是从 $(\mathcal{W}, \text{op}, \text{context})$ 到 $\{\text{allow}, \text{deny}, \text{escalate}\}$ 的可计算函数。

**定义 6.2（价值对齐—可操作含义）**  
对齐不是抽象「好」，而是：

1. **结构对齐**：训练目标使模型在分布上偏好「良构 + 有证据」输出（SFT/DPO）。  
2. **行为对齐**：$\mathcal{P}$ 使高风险 `ACT` 必须 `escalate` 或 `deny`。  
3. **现实对齐**：KPI（成功率、违规率）进入 `OBSERVE` 后的 **再训练或阈值调整** 闭环。

**公理 L4-A1（单调审计）**  
凡改变 $O$ 的 `ASSERT/MERGE` 必须产生 **append-only 审计记录**（谁、何时、凭何证据）。

**实现对应**：项目 `configs/`、`G` 层治理规则；`WorldModelTrainingSpec`；外部 IAM/审批流。

---

## 七、L5 计算逻辑：度量—优化—校准

**定义 7.1（核心度量族）** — 至少选其二作为产品 KPI：

| 符号 | 含义 | 用途 |
|------|------|------|
| $\mathcal{M}_{\text{val}}$ | 校验失败率、冲突数 | 质量门槛 |
| $\mathcal{M}_{\text{pred}}$ | 预测对数似然 / 命中率 | 压缩智能、动态可信 |
| $\mathcal{M}_{\text{cal}}$ | 置信分桶校准误差 | 防止「假确定」 |
| $\mathcal{M}_{\text{ground}}$ | 有外部锚断言占比 | 接地程度 |

**定理 7.1（闭环可收敛的必要条件—工程版）**  
若仅有 $\mathcal{M}_{\text{val}}$ 而无 $\mathcal{M}_{\text{pred}}$ 或 $\mathcal{M}_{\text{ground}}$，则系统可「内部自洽但与现实脱钩」；若仅有后两者而无 $\mathcal{M}_{\text{val}}$，则易结构腐烂。**产品化应同时暴露两类看板。**

**实现对应**：`ComprehensiveModelTuner.score` 扩展；日志管道；CI golden IR。

---

## 八、演绎链示例：从公理到「必须实现的接口」

1. **L0-A1 + L1** ⇒ 必须实现：**引用完整性检查** + **主观→客观 id 解析**。  
2. **L3-A1** ⇒ 必须区分：**推理服务 API** 与 **写库 API**，且默认不同权限。  
3. **L4-A1** ⇒ 必须：**审计日志** 或与事件溯源存储等价物。  
4. **L5** ⇒ 必须：**评测作业**（非仅单元测试），周期性跑在 hold-out 现实回放数据上。

此为「严密」之处：不是口号，而是**缺接口即违反已采纳公理**。

## 九、产品化模型：能力与制品矩阵

| 堆栈层 | 产品能力（用户可感知） | 工程制品（对内） |
|--------|------------------------|------------------|
| L1 | 图谱/时间线/多视图一致 | `ViewSpec`、Schema、图存储 |
| L2 | 来源可点击、置信可见、多视角切换 | 证据面板、租户隔离配置 |
| L3 | 仿真/what-if、工作流执行、反馈看板 | 运行时适配器、事件总线 |
| L4 | 审批、权限、合规报表 | 策略引擎、审计查询 API |
| L5 | 质量分、接地率、漂移告警 | 指标管道、Dashboard |

**最小可行产品（MVP）闭合路径**：INGEST → 良构 WM → 至少一态投影 → MERGE 人审 → OBSERVE 记反馈 → $\mathcal{M}_{\text{val}}$ 周报。  
**增强路径**：加 PREDICT + $\mathcal{M}_{\text{pred}}$；加 DPO + $\mathcal{M}_{\text{cal}}$；加多租户 + L2 视角。

## 十、成熟度模型 WM-CMM（建议）

| 等级 | 特征 | 关键判据 |
|------|------|----------|
| **1 结构级** | 良构 IR 可存可取 | L1 全绿 |
| **2 接地级** | 比例化外部证据、可回放摄取 | $\mathcal{M}_{\text{ground}}$ 达标 |
| **3 动态级** | 预测与执行分离、闭环 OBSERVE | L3+L4 跑通 |
| **4 对齐级** | 偏好训练 + 校准 + 策略联动 KPI | L5 全链路 |

用于销售/立项/技术债排期同一套语言。
