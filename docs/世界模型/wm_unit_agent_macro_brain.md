# 微观脑与宏观脑：Unit Agent 世界模型范式

> **命题**：LLM 的「智能涌现」范式对世界模型有强参考性，但**尺度与原子不同**——LLM 是 **微观脑**（原子为 token），世界模型宜建模为 **宏观脑**（原子为 **Unit Agent / 认知体**）。本文从该对比**严格推演**体系原理与实现路径，并与 [`wm_system_logic_and_product_stack.md`](./wm_system_logic_and_product_stack.md)（WM-SLPS）、[`wm_theory.md`](./wm_theory.md) 对齐。

---

## 一、核心对照：微观脑 vs 宏观脑

| 维度 | 微观脑（LLM） | 宏观脑（世界模型 WM） |
|------|----------------|------------------------|
| **原子** | Token $t \in \mathcal{V}$ | **Unit** $u$：具边界与身份的认知体（可映射人、组织、机器人、数字代理、抽象角色） |
| **局部结构** | 上下文窗口内注意力形成的瞬时关联 | **Unit 内**稳定结构：局部意图、信念、能力、状态 |
| **全局结构** | 参数中压缩的统计规律 | **Unit 间**图：关系边 + 事件 + 时空锚（五类关系等） |
| **涌现** | 规模/数据过阈后能力相变 | **网络级**涌现：多 Unit 耦合后出现可协调的宏观秩序（市场、组织行为、系统韧性） |
| **闭环** | 训练-推理循环；无身体时缺物理闭合 | 每 Unit 可有 **感知→建模→意图→行动→反馈**；系统级再加 **编排与治理** |
| **接地** | 依赖训练语料分布 | 依赖 **Unit 与真实 id/传感器/业务库** 的锚定 |

**根本洞察**：世界模型不是「把全世界压进一个 Transformer」，而是 **在宏观尺度上重放「多认知体 + 关系 + 事件」的物理学**；LLM 适合作为 **Unit 内部的微观推理引擎** 或 **全局抽取/自然语言接口**，而非替代 Unit 本体地位。

## 二、形式化：Unit Agent（认知体）

**定义 2.1（Unit）**  
Unit 是有身份 $id(u)$ 的个体，在世界模型中占据**一个可引用节点**，且携带**最小认知闭包**字段集合：

$$
u = \bigl( id,\; B_u,\; K_u,\; G_u,\; A_u,\; C_u,\; \partial u \bigr)
$$

| 符号 | 含义 | 与 `wm_theory` / IR 的对应 |
|------|------|---------------------------|
| $B_u$ | **信念/局部世界切片**（对邻域事实的立场与置信） | `CognitionIR.beliefs` + 链接到客观实体/事件 |
| $K_u$ | **能力注册**（能执行的操作类型：读库、发单、对话、规划…） | `EntityIR.capabilities` + 策略层 `Policy` |
| $G_u$ | **意图与约束**（目标、优先级、合规边界） | `IntentionIR`（可挂 `about_entity_ids`） |
| $A_u$ | **行动历史/计划前沿** | `PracticeIR` |
| $C_u$ | **反馈通道**（误差、奖励、人类纠正） | `FeedbackIR` |
| $\partial u$ | **边界**（合法输入输出接口、可观测邻域） | 工程上：API scope、数据权限、传感器集合 |

**公理 2.1（认知闭包）**  
凡声称「某 Unit 具备完整认知闭环」，须在系统内可实现：**$\partial u$ 上可观测 → 可写入 $C_u$ → 可更新 $B_u/G_u$ → 可经 $K_u$ 发出 $A_u$**，且无悬空引用（见 WM-SLPS L0-A1）。

**注**：非智能实体（石头、纯数据行）可退化为 **$K_u = \emptyset$** 的 Unit，仅参与关系与事件，不强制五要素齐全。

## 三、链接即关系：宏观连通性

**定义 3.1（Unit 间链接）**  
任意有序对 $(u_i, u_j)$ 上的链接 $l$ 是 `RelationType` 中某一类的实例（及可选事件锚、时间），即现有 **五类关系** 的语义：邻接、因果、部分-整体、影响、角色。

**命题 3.1（宏观脑 = Unit 图 + 事件动力学）**  
宏观世界模型 $\mathcal{W}$ 的客观层 $O$ **优先解释为**：

$$
O \;\equiv\; (U,\; L,\; E,\; ST)
$$

- $U$：Unit 集合（与 Entity 集合同构或为其超集）  
- $L$：Unit 间链接（即 `RelationIR`）  
- $E$：事件（改变状态、创建链接、触发行动）  
- $ST$：时空锚

主观层 $S$ 则可分解为 **各 Unit 上 $G,B,A,C$ 的聚合**（及可选全局 orchestrator 的主观块）。

## 四、双尺度涌现：如何与 LLM 范式衔接

| LLM 涌现（微观） | 世界模型涌现（宏观） |
|------------------|----------------------|
| 参数与数据过阈 → CoT、工具使用等 | Unit 数、链接密度、事件吞吐量过阈 → **协调模式**（分工、层级、市场清算） |
| Attention 形成 token 级「意义网络」 | **关系 + 事件**形成 Unit 级「社会-系统网络」 |
| 后训练压缩「偏好秩序」 | **策略门控 + 多主体偏好**压缩为可执行规范（WM-SLPS L4） |

**实践含义**：微观上可用 **SFT/DPO** 让 LLM 更好生成 **IR 或 Unit 补丁**；宏观上要单独设计 **网络度量**（如跨 Unit 一致性、协调延迟、级联故障风险），二者 KPI 不可混用。

## 六、实现路径（分阶段，可验收）

### 阶段 A：Unit 与 IR 对齐（结构级）

- 将 **EntityIR** 明确定位为 **Unit 载体**：`capabilities` ↔ $K_u$；扩展或关联表承载 $G,B,A,C$（或嵌入 `SubjectiveSystem` 的按-Unit 分片）。  
- **关系边**仅允许端点为 Unit id（或事件 id），跑 WM-SLPS L1 校验。  
- 交付：良构图 + 单一 Unit 类型文档（schema）。

### 阶段 B：单 Unit 闭环（认知级）

- 为每类 Unit 定义 **$\partial u$**：输入源列表、可执行动作枚举、反馈写回路径。  
- 实现最小循环：**OBSERVE → 更新 $B_u,C_u$ →（可选 LLM）更新 $G_u$ → ACT → 事件 $E$**。  
- LLM 角色：**$u$ 内部的微观脑**，输出结构化补丁而非自由文本。  
- 交付：至少一类 Unit（如「数字员工代理」）端到端 demo + 审计日志。

### 阶段 C：多 Unit 编排（宏观级）

- 引入 **Orchestrator**（可为独立 Unit 或无 $K$ 的服务）：调度、冲突消解、全局意图分解。  
- 链接类型驱动交互协议（如 `Role` 定义汇报链，`Affect` 触发下游重规划）。  
- 交付：多 Unit 仿真或沙箱 + 网络级指标（见 WM-SLPS L5 扩展）。

### 阶段 D：训练与对齐双轨

- **微观轨**：继续用偏好学习优化 **IR/补丁** 质量。  
- **宏观轨**：用 **日志+仿真** 优化 **策略 $\mathcal{P}$** 与 Unit 能力配置（何 Unit 可自动 ACT）。  
- 交付：分离的评测看板，避免用 loss 代替协调成功率。

## 七、与现有代码的映射（当前仓库）

| 概念 | 当前落点 | 演进方向 |
|------|-----------|----------|
| Unit 身份 | `EntityIR.id` | 显式 `unit_kind` / 生命周期状态 |
| 能力 $K_u$ | `EntityIR.capabilities` | 与 L4 策略表绑定（可执行名 → 权限） |
| 闭包 $G,B,A,C$ | 全局 `SubjectiveSystemIR` | **按 Unit 分片**或 `metadata.unit_id` 关联 |
| 链接 | `RelationIR` + `RelationType` | 不变，强化端点必须为 Unit |
| 运行时 | `CognitivePracticeRuntime` | 扩展为 **per-unit runtime** 或注册表 |
| 投影 | `CognitiveMode` | 增加「以 Unit 为中心」的子图投影 |

## 八、一句话公理（便于对内对齐）

**宏观脑世界模型 = 由 Unit Agent（认知体）作为原子节点构成的图，节点内嵌最小认知闭环，边为关系与事件所定义的社会-物理连通；LLM 是 Unit 内或接口上的微观脑，负责高熵语言与候选结构，秩序与真值由图、证据与门控共同裁定。**
