## 二、知识表示：图谱 vs 向量 vs 符号

### 2.1 方案对比

| 方案 | 核心机制 | 优势 | 劣势 | 适用场景 |
|------|----------|------|------|----------|
| **知识图谱** | 实体-关系-属性图 | 可解释、推理、关联发现 | 构建成本高、扩展慢 | 领域知识、推理任务 |
| **向量嵌入** | 稠密向量表示 | 语义匹配、泛化强 | 不可解释、黑箱 | 语义检索、NLP任务 |
| **符号逻辑** | 形式化规则 | 可证明、可审计 | 覆盖度低、脆弱 | 高安全、可解释场景 |
| **混合表示** | 图谱+向量+规则 | 互补增强 | 复杂度高 | 复杂推理 |

## 三、推理机制：LLM推理 vs 图谱推理 vs 规划驱动

### 3.1 方案对比

| 方案 | 核心机制 | 优势 | 劣势 | 代表工作 |
|------|----------|------|------|----------|
| **LLM推理** | CoT / ToT / ReAct | 通用、灵活 | 不可控、幻觉 | OpenAI, Anthropic |
| **图谱推理** | 路径遍历 / 嵌入推理 | 可解释、精准 | 覆盖度依赖KG | GNN, KGQA |
| **规划驱动推理** | 世界模型 + 模拟推演 | 长程规划、目标导向 | 计算密集 | SWAP, SimuRA |
| **混合推理** | LLM + KG + 规划 | 互补增强 | 复杂度高 | VAGEN |

### 3.2 规划驱动推理深度分析

**核心思想**：用世界模型进行"模拟推演"，而非直接输出行动

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Planning via World Model                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   传统推理 (LLM):                                                       │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐                            │
│   │  State │───→│  Think  │───→│ Action  │                            │
│   └─────────┘    └─────────┘    └─────────┘                            │
│                        │                                                │
│                        ▼                                                │
│   规划驱动推理:                                                         │
│   ┌─────────┐    ┌──────────────┐    ┌──────────────┐    ┌─────────┐ │
│   │  State │───→│ Simulate(H)   │───→│ Evaluate     │───→│ Action  │ │
│   │   s    │    │ 想象轨迹      │    │ 评估成本     │    │    a   │ │
│   └─────────┘    └──────────────┘    └──────────────┘    └─────────┘ │
│                                                                         │
│   关键区别:                                                             │
│   - 传统: s → a (直接映射)                                              │
│   - 规划: s → 模拟多步轨迹 → 选最优 → a                                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**SWAP架构** (ACL 2025):
```
Policy Model → 生成计划 H
World Model  → 预测状态 s_{t+1}
Controller   → 决定继续/终止
```

**SimuRA架构**:
```
┌─────────────────────────────────────────────────────────────────────────┐
│                     SimuRA: 模拟推理架构                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   输入: Goal + State                                                    │
│        │                                                                │
│        ▼                                                                │
│   ┌─────────────────────────────────────────┐                          │
│   │       LLM World Model                   │                          │
│   │  - 提取关键信息 → 自然语言latent space  │                          │
│   │  - 模拟下一步状态                       │                          │
│   └─────────────────────────────────────────┘                          │
│        │                                                                │
│        ▼                                                                │
│   ┌─────────────────────────────────────────┐                          │
│   │       Cost Minimization                 │                          │
│   │  - 评估模拟轨迹与目标距离               │                          │
│   │  - 选择成本最低的路径                   │                          │
│   └─────────────────────────────────────────┘                          │
│        │                                                                │
│        ▼                                                                │
│   Action Output                                                         │
│                                                                         │
│   效果: 航班搜索成功率 0% → 32.2%                                      │
│                                                                         │
```

### 3.3 选型建议

| 场景 | 推荐方案 |
|------|----------|
| **短程推理** | LLM CoT |

| **长程规划** | 规划驱动推理 (SimuRA) |
| **复杂任务** | VAGEN模式：LLM+世界模型+状态估计 |

## 四、神经网络潜在空间规划 vs 符号规划

### 4.1 方案对比

| 方案 | 核心机制 | 优势 | 劣势 | 代表工作 |
|------|----------|------|------|----------|
| **潜在空间规划** | 隐空间模拟+优化 | 样本高效、泛化强 | 不可解释 | Dreamer系列 |
| **符号规划** | 搜索/逻辑求解 | 可解释、精确 | 脆弱 | STRIPS, PDDL |
| **混合规划** | 符号接口+神经内核 | 互补 | 界面设计难 | Neuro-Symbolic |

## 五、LLM as World Model Builder

### 5.1 核心范式

**与Dreamer的区别**：
- Dreamer: 从数据学习隐空间世界模型
- LLM as Builder: 用LLM构建/编译世界模型

```
┌─────────────────────────────────────────────────────────────────────────┐
│               LLM as World Model Builder                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   核心理念:                                                             │
│   ┌─────────────────────────────────────────────────────────────────┐ │
│   │  LLM不直接模拟世界，而是构建世界的代码化表示                      │ │
│   │  (transition function as Python program)                        │ │
│   └─────────────────────────────────────────────────────────────────┘ │
│                                                                         │
│   WorldCoder架构:                                                       │
│                                                                         │
│   ┌───────────┐    ┌─────────────┐    ┌─────────────┐                  │
│   │ Experience│───→│ Program     │───→│ Transition  │                  │
│   │   Data   │    │ Synthesizer │    │   Function  │                  │
│   └───────────┘    └─────────────┘    └─────────────┘                  │
│                           │                                                │
│                           ▼                                                │
│                    ┌─────────────┐                                        │
│                    │  Planner    │                                        │
│                    │ (使用WM规划)│                                        │
│                    └─────────────┘                                        │
│                                                                         │
│   关键创新:                                                             │
│   1. 程序化世界模型: 用Python代码表达T(s'|s,a)                         │
│   2. 知识迁移: 程序片段可复用                                           │
│   3. 乐观探索: "Zone of Proximal Development" 驱动探索                   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.2 技术路线对比

| 路线 | 机制 | 优势 | 劣势 | 代表 |
|------|------|------|------|------|
| **隐空间学习** | 神经网络拟合 | 端到端、泛化 | 不可解释 | Dreamer |
| **代码生成** | LLM写程序 | 可解释、可迁移 | 依赖LLM能力 | WorldCoder |
| **检索增强** | RAG + 规则 | 动态知识 | 上下文限制 | R-WoM |
| **对齐修正** | 规则覆盖LLM | 纠错能力强 | 规则构建难 | World Alignment |

### 5.3 LLM世界模型类型

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     LLM世界模型类型                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   1. 直接模拟 (Direct Simulation)                                       │
│      LLM(s, a) → s'  (ReAct风格)                                       │
│      局限: 长程一致性、幻觉                                              │
│                                                                         │
│   2. 检索增强 (Retrieval-Augmented)                                     │
│      LLM(s, a, retrieved_knowledge) → s'                               │
│      优势: 缓解幻觉、接入实时知识                                        │
│      代表: R-WoM, OSWorld                                               │
│                                                                         │
│   3. 符号规则增强 (Neuro-Symbolic)                                      │
│      P_world = δ[rule(s,a)] ? rule : LLM(s,a)                          │
│      优势: 规则覆盖LLM错误                                               │
│      代表: World Alignment                                              │
│                                                                         │
│   4. 程序化构建 (Program Synthesis)                                      │
│      LLM → Python(s',s,a) → 编译执行                                   │
│      优势: 可解释、可迁移                                                │
│      代表: WorldCoder                                                   │
│                                                                         │
│   5. 假设推理 (Hypothesis Reasoning)                                     │
│      LLM生成"理论" → Bayesian推断 → 验证                                │
│      优势: 可探索因果机制                                                │
│      代表: WorldLLM                                                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 七、总结

### 7.1 技术选型结论

| 维度 | 当前方案 | 目标方案 | 理由 |
|------|----------|----------|------|
| **知识表示** | 静态JSON | 混合图谱+向量+规则 | 平衡可解释与泛化 |
| **推理机制** | 规则匹配 | 规划驱动+LLM | 长程规划能力 |
| **世界建模** | 配置驱动 | LLM程序化 | 可迁移、可解释 |
| **学习演化** | 简化反馈 | 知识图谱+程序更新 | 持续演化能力 |

### 7.2 核心差异

| 路线 | 定位 | 适合场景 |
|------|------|----------|
| **Dreamer** | 隐空间学习 | 机器人、视觉控制 |
| **WorldCoder** | 代码化构建 | 可解释、迁移重要 |
| **SimuRA** | 模拟推理 | 复杂规划任务 |
| **VAGEN** | 世界模型训练 | 视觉Agent |
| **UAS WM** | 认知架构 | 业务智能体 |
